# [NES(Never Ending Story)](http://nes-env.eba-9ycvw3yi.ap-northeast-2.elasticbeanstalk.com/)
예술 대학생을 위한 졸업 작품 중개 서비스

## 🧑🏻‍🎓 기획 취지
NES팀은 '젊은 예술가들이 직면할 수 밖에 없는 작품의 전시와 그에 따른 비용에 관한 문제를 어떻게 해소시켜 줄 수 있을까?' 고민하였습니다. <br />

'끝나지 않는 이야기’ 라는 뜻을 담고 있는 팀 이름은 위와 같은 고민으로부터 출발했습니다. <br />

젊은 예술가들의 어려움에 공감하며 그들의 꿈을 이루는 것을 돕고자 졸업 작품 중개 서비스를 기획하였습니다. <br />

## 🧑🏻‍🎓 KPT 회고
### 🧡 Keep (앞으로도 유지했으면 하는 좋은 점)
- 김예린 : 다 같이 팀원들과 상의하면서 기능과 화면을 하나씩 꾸려나갔던 점이 가장 좋았습니다. 제가 한가지의 생각에만 매몰되어 있을 때 팀원들에게 물어보고 의견을 나누며 생각을 확장할 수 있는 진정한 협업을 경험한 것이 이번 프로젝트에서 가장 좋은 점이라고 생각합니다. 경청해주는 팀원들을 보며 저도 협업에 대한 자세를 다시 배울 수 있었습니다. 

- 문현동 : 프로젝트를 진행하며 신기술도 많이 배우고 실력이 늘어나는게 느껴질정도여서 정말 좋았습니다. 

- 백솔비 : 깃허브 칸반을 이용해 체계적으로 진행할 수 있어서 좋았습니다. 처음에는 아침 데일리 스크럼만 진행하다가 저녁에도 오늘 무엇을 했는지, 어느 부분이 안 되었는지 이야기하는 시간을 가졌습니다. 확실히 두번의 스크럼을 진행하니 명확하게 어디가 문제고 어디를 해야할 지 알 수 있어서 좋았습니다. 

- 이명학 : 이번에 프론트엔드와 백엔드를 구분해 프로젝트를 진행해보니, 서로 소통하고 협업하는것의 중요성을 깨닫게 됐습니다. 또한 프로젝트를 진행하면서, 이전에 사용해보지 않았던 API들을 사용해보고, 깃허브 칸반등을 이용해본 점도 좋은 개발과정에서 경험이었던 것 같습니다. 

- 최근영 : 뭔가 새로운 기능을 중시하기 보다는 기존의 기능들을 프론트 쪽으로 꾸며볼 수 있었고 다양한 JS 기능들과 CSS를 많이 만들어봤으며 팀원들이 개개인 능력이 뛰어나서 정말 편하게 프로젝트를 진행해볼 수 있었습니다. 추가적인 기능들을 좀 더 구현해볼 기회가 있었으면 좋겠습니다. 

### 💙 Problem (고쳐졌으면 하는 문제점이나 아쉬운 점)
- 김예린 : 처음부터 기능과 모델에 대한 탄탄한 계획을 가지고 구축을 했어야 했는데 시간이 지나고 계속해서 모델을 수정하는 것이 아쉬웠습니다. 모델 기획에 대한 중요성을 깨달았습니다.

- 문현동 : 스크롤 바를 사용하기 위해 이것저것 노력을 많이 기울였는데 스크롤바가 뜻대로 작동하는 경우가 거의 없거나 또는 그렇게 하기 위해서 상당히 많은 공을 들여야 했습니다.

- 백솔비 : 팀원들의 모니터 크기에 따라 배치가 달라져서 아쉬웠습니다.

- 이명학 : 이번에 회원쪽을 맡으면서, 서비스의 기초가 되는 회원관리쪽의 중요성을 깨달았습니다. 일반 로그인과 카카오 로그인시 유저를 어떻게 DB에 저장할 것인지 설계하는게 중요하다는 걸 깨달았고, 사이트가 방대해질수록 회원관리를 섬세하게 해야된다는 점을 깨달았습니다.

- 최근영 : 화면 구성하는 부분에 있어서 개개인 모니터 크기를 맞추는 것이 좀 힘들었고 나머지 부분은 딱히 문제는 없었던 것 같습니다.

### 🧡 Try (문제점을 해결하기 위해 시도해볼 수 있는 해결책)
- 김예린 : 다음 프로젝트에서는 시간을 가지고 기능과 모델에 대한 기획표를 작성하여 한번 더 전체적인 데이터베이스에 대한 흐름을 생각해봤으면 좋겠습니다.

- 문현동 : 스크롤바 문제를 해결하기 위해 여러 문서를 찾아봤고 비슷한 문제에 대해 공식문서와 스택오버플로우 답변들을 참조하여 해결했습니다.

- 백솔비 : 팀원들의 모니터 사이즈를 수집하고 이를 하나하나 맞춰가면서 다시 수정했습니다. 앞으로는 모니터 크기에 구애받지 않는 반응형 디자인을 시도해야겠습니다. 변수로 만들어서 계산하는 방식으로 css를 짜면 해결될 수 있다고 생각합니다.

- 이명학 : 일반유저와 로그인 유저를 분류하기 위해 test라는 필드를 추가했는데, splite에서는 username을 비워도 되었으나, postgreSQL에서는 비우면 안된다는 점을 깨달았다. 당연하게 될 거라고 생각했던 부분에서 뜻밖의 오류가 발생하고 나니 설계를 잘 해야겠다는 생각이 들었습니다.

- 최근영 : 모니터 사이즈 문제 외에도 뭔가 화면을 구성하는데 시간소요가 너무 많이 걸렸던 것 같고 생각보다 완성이 늦게돼서 다른 기능을 도와주는데 막혔던 것 같습니다.
